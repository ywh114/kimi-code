# Fork Patches

Short log of patches applied on top of upstream `kimi-code`.

| Patch | Description | Files |
|-------|-------------|-------|
| Remove tips | Disabled rotating toolbar tips above the input box and tips next to the streaming spinner. | `src/tui/components/chrome/footer.ts`, `src/tui/components/chrome/header.ts` |
| Live-thinking expansion (Ctrl+Y) | `Ctrl+Y` expands/collapses reasoning while it is streaming; finalized blocks are unaffected. Expanded live thinking uses a static bullet instead of the animated Braille spinner to stop flashing. `Ctrl+E` is left free for end-of-line. | `src/tui/components/messages/thinking.ts`, `src/tui/components/editor/custom-editor.ts`, `src/tui/components/dialogs/help-panel.ts` |
| Finalize thinking before tool calls | Live thinking blocks are finalized before a tool call renders, so tool output updates do not fight with expanded thinking. | `src/tui/controllers/session-event-handler.ts` |
| Footer badges | Added `xtool` and `xthink` indicators next to `yolo`/`plan`. | `src/tui/types.ts`, `src/tui/kimi-tui.ts`, `src/tui/controllers/streaming-ui.ts`, `src/tui/components/chrome/footer.ts` |
| Symbols / spinners | Status bullet changed from `●` to legacy `•`. Braille/moon spinners replaced with ASCII `| / - \\` to avoid missing-glyph boxes. | `src/tui/constant/symbols.ts`, `src/tui/constant/rendering.ts` |
| Sticky shell mode (`Ctrl+X`) | `Ctrl+X` toggles bash mode; submits stay in shell mode until toggled off. | `src/tui/controllers/editor-keyboard.ts`, `src/tui/components/editor/custom-editor.ts` |
| Bash prompt symbol | Bash-mode prompt changed from `!` to `$`. | `src/tui/components/editor/custom-editor.ts` |
| `!!` direct shell eval | `!!<cmd>` runs Bash directly via `session.runShellCommand`, streams output to a `SHELL` panel, prompt shows `&`, output is deduplicated, and `Esc` closes the panel. | `src/tui/components/editor/custom-editor.ts`, `src/tui/kimi-tui.ts`, `src/tui/controllers/shell-eval-panel.ts`, `src/tui/components/panes/shell-eval-panel.ts` |
| `/help` shortcuts | Added `Ctrl+Y` (live thinking), `Ctrl+X` (shell mode), and `Ctrl+R` (reverse-i-search) to the `/help` keyboard list, and corrected the `Esc` description. | `src/tui/components/dialogs/help-panel.ts` |
| Reverse-i-search (`Ctrl+R`) | `Ctrl+R` starts a reverse history search; repeated `Ctrl+R` cycles older matches, `Enter` accepts, `Esc`/`Ctrl+C`/`Ctrl+G` cancel. The search query is edited in the input line with a `?` prompt, and the matched history entry is shown as a dim hint. All search updates force a full redraw so stale frames are not left behind while the transcript scrolls. | `src/tui/components/editor/custom-editor.ts`, `packages/pi-tui/src/components/editor.ts` |
| `/context-max` command | New slash command to cap the effective context window of the current model: `/context-max 256k` (also `128k`, `512k`, `1M`, raw numbers), `/context-max reset`, and no-arg status. Writes `models.<alias>.overrides.maxContextSize` to config.toml (preserved across managed catalog refreshes); the running session picks it up immediately because `ProviderManager` reads config live, and the footer gauge is refreshed from `session.getStatus()`. Auto-compaction then triggers at ~85% of the cap (minus the 50k reserve), lowering per-turn input-token spend on long sessions. | `src/tui/commands/config.ts`, `src/tui/commands/registry.ts`, `src/tui/commands/dispatch.ts`, `src/tui/commands/index.ts` |
| Force truecolor | `chalk.level = 3` when stdout is a TTY. chalk auto-detects level 2 when `COLORTERM` is unset (e.g. inside tmux with `TERM=screen/tmux-256color`), flattening custom theme hexes into muddy 256-cube approximations (thinking `#928374` gray became khaki). | `src/tui/theme/theme.ts` |
| `Esc` vs `Ctrl-C` | `Esc` only closes dialogs/panels and double-tap undo; it no longer cancels streaming/compaction/`/init`. `Ctrl-C` handles all interruption. Error messages that suggested `Esc or Ctrl-C` now say `Ctrl-C`. | `src/tui/controllers/editor-keyboard.ts`, `src/tui/kimi-tui.ts`, `src/tui/commands/resolve.ts`, `src/tui/commands/config.ts`, `src/tui/commands/undo.ts` |
| Subagent model selection (`[subagent.models]`) | Per-profile model overrides for subagents: `[subagent.models]` in config.toml maps profile names (`coder`, `explore`, `plan`) to aliases from `[models]`, e.g. `coder = "kimi-for-coding"` to offload coding loops to a cheaper model while the main session plans on a stronger one. Spawn, resume, and retry resolve via `resolveSubagentModelAlias`; profiles without an entry inherit the parent model, and an entry naming an unconfigured alias falls back to the parent model with a warning. Applies to `Agent`/`AgentSwarm`, foreground and background; `/btw` side questions always use the parent model. Read from the session-bound config snapshot, so edits take effect for sessions started after the change. | `packages/agent-core/src/config/schema.ts`, `packages/agent-core/src/session/subagent-host.ts` |
| Delete sessions in `/sessions` | `Ctrl+K` in the session picker arms a delete for the selected session (`Delete "<title>"? Ctrl+K confirm · Esc cancel`); a second `Ctrl+K` confirms, any other key cancels, and the current session is refused. Deletion is real, not archive: a new `deleteSession` RPC (closes the session first if it is open) calls `SessionStore.delete`, which removes the session directory and its `session_index.jsonl` lines via a queue-serialized temp-file rewrite. After deletion the picker refreshes and moves the selection to the neighboring session. | `packages/agent-core/src/session/store/session-index.ts`, `packages/agent-core/src/session/store/session-store.ts`, `packages/agent-core/src/rpc/core-api.ts`, `packages/agent-core/src/rpc/core-impl.ts`, `packages/node-sdk/src/rpc.ts`, `packages/node-sdk/src/kimi-harness.ts`, `src/tui/components/dialogs/session-picker.ts`, `src/tui/kimi-tui.ts` |
| Tests | Updated tests for the above changes. | `test/tui/controllers/editor-keyboard.test.ts`, `test/tui/kimi-tui-message-flow.test.ts`, `test/tui/components/editor/custom-editor.test.ts`, `test/tui/components/panels/help-panel.test.ts`, `test/tui/commands/context-max.test.ts`, `packages/agent-core/test/session/subagent-host.test.ts`, `packages/agent-core/test/config/configs.test.ts`, `packages/agent-core/test/session/store.test.ts`, `test/tui/components/dialogs/session-picker.test.ts` |
